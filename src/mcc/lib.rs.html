1//! Michael's C Compiler — core library
2//!
3//! This crate implements the core compilation pipeline and data structures for
4//! a small, testable C compiler. It is designed to be embedded in tools and
5//! tests, and powers the `mcc` command-line tool in the `mcc-driver` crate.
6//!
7//! The architecture follows a classic pipeline, with each stage tracked by
8//! [`salsa`] to enable incremental recomputation and diagnostics accumulation:
9//!
10//! - Preprocessing: [`preprocess`]
11//! - Parsing: [`parse`]
12//! - Lowering to TAC: [`lowering::lower`]
13//! - Code generation (ASM IR): [`codegen::generate_assembly`]
14//! - Rendering (assembly text): [`render::render_program`]
15//! - Assembling and linking: [`assemble_and_link`]
16//!
17//! Data is passed through well-defined types like [`types::SourceFile`],
18//! [`types::Ast`], [`codegen::asm::Program`], and [`Text`]. Diagnostics are
19//! accumulated via a salsa accumulator, see [`diagnostics`]. Targets are
20//! described using [`target_lexicon::Triple`].
21//!
22//! Quick start
23//! -----------
24//! Parse, lower, generate, and render a tiny program to assembly text:
25//!
26//! ```rust
27//! use mcc::{Database, SourceFile, Text};
28//!
29//! let db = Database::default();
30//! let src = "int main(void) { return 0; }";
31//! let file = SourceFile::new(&db, Text::from("main.c"), Text::from(src));
32//!
33//! // Parse → TAC → ASM IR → assembly text
34//! let ast = mcc::parse(&db, file);
35//! let tacky = mcc::lowering::lower(&db, ast, file);
36//! let asm_ir = mcc::codegen::generate_assembly(&db, tacky);
37//! let asm_text = mcc::render_program(&db, asm_ir, mcc::default_target()).unwrap();
38//!
39//! assert!(asm_text.as_str().contains("main"));
40//! ```
41//!
42//! Capturing diagnostics
43//! ---------------------
44//! Each stage can emit diagnostics which are accumulated rather than panicking.
45//! You can retrieve them using the stage’s `accumulated::<T>()` helper:
46//!
47//! ```rust
48//! use mcc::{Database, SourceFile, Text, diagnostics::Diagnostics};
49//!
50//! let db = Database::default();
51//! let file = SourceFile::new(&db, "test.c".into(), "int main(void) {}".into());
52//! let _ = mcc::parse(&db, file);
53//! let diags: Vec<&Diagnostics> = mcc::parse::accumulated::<Diagnostics>(&db, file);
54//! // Render with codespan-reporting using `mcc::Files`
55//! ```
56//!
57//! Targets and OS-specific behavior
58//! --------------------------------
59//! Rendering takes a [`target_lexicon::Triple`]. Use [`default_target()`] for a
60//! reasonable default. On macOS, symbol names are rendered with a leading
61//! underscore (e.g., `_main`); on Linux, a `.note.GNU-stack` section is emitted.
62//!
63//! Notes on preprocessing
64//! ----------------------
65//! [`preprocess`] invokes the system C compiler’s preprocessor (e.g. `cc -E -P`).
66//! The `mcc` driver currently runs preprocessing and writes the result to a
67//! temporary file; the parser reads the original [`SourceFile`] contents.
68//!
69//! See also
70//! --------
71//! - [`codegen::asm`] for the assembly IR
72//! - [`diagnostics`] for diagnostics accumulation and error codes
73//! - [`mcc-driver`] for CLI orchestration and staged callbacks

123impl Debug for Database {
124    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
125        let Database { storage: _ } = self;
126
127        f.debug_struct("Database").finish_non_exhaustive()
128    }
129}

131pub fn default_target() -> Triple {

134        ..Triple::host()
