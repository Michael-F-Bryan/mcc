1//! Cycle handling
2//!
3//! Salsa's default cycle handling is quite simple: if we encounter a cycle (that is, if we attempt
4//! to execute a query that is already on the active query stack), we panic.
5//!
6//! By setting `cycle_fn` and `cycle_initial` arguments to `salsa::tracked`, queries can opt-in to
7//! fixed-point iteration instead.
8//!
9//! We call the query which triggers the cycle (that is, the query that is already on the stack
10//! when it is called again) the "cycle head". The cycle head is responsible for managing iteration
11//! of the cycle. When a cycle is encountered, if the cycle head has `cycle_fn` and `cycle_initial`
12//! set, it will call the `cycle_initial` function to generate an "empty" or "initial" value for
13//! fixed-point iteration, which will be returned to its caller. Then each query in the cycle will
14//! compute a value normally, but every computed value will track the head(s) of the cycles it is
15//! part of. Every query's "cycle heads" are the union of all the cycle heads of all the queries it
16//! depends on. A memoized query result with cycle heads is called a "provisional value".
17//!
18//! For example, if `qa` calls `qb`, and `qb` calls `qc`, and `qc` calls `qa`, then `qa` will call
19//! its `cycle_initial` function to get an initial value, and return that as its result to `qc`,
20//! marked with `qa` as cycle head. `qc` will compute its own provisional result based on that, and
21//! return to `qb` a result also marked with `qa` as cycle head. `qb` will similarly compute and
22//! return a provisional value back to `qa`.
23//!
24//! When a query observes that it has just computed a result which contains itself as a cycle head,
25//! it recognizes that it is responsible for resolving this cycle and calls its `cycle_fn` to
26//! decide how to do so. The `cycle_fn` function is passed the provisional value just computed for
27//! that query and the count of iterations so far, and must return either
28//! `CycleRecoveryAction::Iterate` (which signals that the cycle head should re-iterate the cycle),
29//! or `CycleRecoveryAction::Fallback` (which signals that the cycle head should replace its
30//! computed value with the given fallback value).
31//!
32//! If the cycle head ever observes that the provisional value it just recomputed is the same as
33//! the provisional value from the previous iteration, the cycle has converged. The cycle head will
34//! mark that value as final (by removing itself as cycle head) and return it.
35//!
36//! Other queries in the cycle will still have provisional values recorded, but those values should
37//! now also be considered final! We don't eagerly walk the entire cycle to mark them final.
38//! Instead, we wait until the next time that provisional value is read, and then we check if all
39//! of its cycle heads have a final result, in which case it, too, can be marked final. (This is
40//! implemented in `shallow_verify_memo` and `validate_provisional`.)
41//!
42//! If the `cycle_fn` returns a fallback value, the cycle head will replace its provisional value
43//! with that fallback, and then iterate the cycle one more time. A fallback value is expected to
44//! result in a stable, converged cycle. If it does not (that is, if the result of another
45//! iteration of the cycle is not the same as the fallback value), we'll panic.
46//!
47//! In nested cycle cases, the inner cycle head will iterate until its own cycle is resolved, but
48//! the "final" value it then returns will still be provisional on the outer cycle head. The outer
49//! cycle head may then iterate, which may result in a new set of iterations on the inner cycle,
50//! for each iteration of the outer cycle.
51

57/// The maximum number of times we'll fixpoint-iterate before panicking.
58///
59/// Should only be relevant in case of a badly configured cycle recovery.
60pub const MAX_ITERATIONS: IterationCount = IterationCount(200);
61
62/// Return value from a cycle recovery function.
63#[derive(Debug)]
64pub enum CycleRecoveryAction<T> {
65    /// Iterate the cycle again to look for a fixpoint.
66    Iterate,
67
68    /// Cut off iteration and use the given result value for this query.
69    Fallback(T),
70}
71
72/// Cycle recovery strategy: Is this query capable of recovering from
73/// a cycle that results from executing the function? If so, how?
74#[derive(Copy, Clone, Debug, PartialEq, Eq)]
75pub enum CycleRecoveryStrategy {
76    /// Cannot recover from cycles: panic.
77    ///
78    /// This is the default.
79    Panic,
80
81    /// Recovers from cycles by fixpoint iterating and/or falling
82    /// back to a sentinel value.
83    ///
84    /// This choice is computed by the query's `cycle_recovery`
85    /// function and initial value.
86    Fixpoint,
87
88    /// Recovers from cycles by inserting a fallback value for all
89    /// queries that have a fallback, and ignoring any other query
90    /// in the cycle (as if they were not computed).
91    FallbackImmediate,
92}

94/// A "cycle head" is the query at which we encounter a cycle; that is, if A -> B -> C -> A, then A
95/// would be the cycle head. It returns an "initial value" when the cycle is encountered (if
96/// fixpoint iteration is enabled for that query), and then is responsible for re-iterating the
97/// cycle until it converges.
98#[derive(Copy, Clone, Debug, PartialEq, Eq, Hash)]
99pub struct CycleHead {
100    pub(crate) database_key_index: DatabaseKeyIndex,
101    pub(crate) iteration_count: IterationCount,
102}
103
104#[derive(Copy, Clone, Debug, PartialEq, Eq, Hash, Default)]
105pub struct IterationCount(u8);
106
107impl IterationCount {
108    pub(crate) const fn initial() -> Self {
109        Self(0)
110    }
111
112    pub(crate) const fn is_initial(self) -> bool {
113        self.0 == 0
114    }
115
116    pub(crate) const fn increment(self) -> Option<Self> {
117        let next = Self(self.0 + 1);
118        if next.0 <= MAX_ITERATIONS.0 {
119            Some(next)
120        } else {
121            None
122        }
123    }
124
125    pub(crate) const fn as_u32(self) -> u32 {
126        self.0 as u32
127    }
128}
129
130/// Any provisional value generated by any query in a cycle will track the cycle head(s) (can be
131/// plural in case of nested cycles) representing the cycles it is part of, and the current
132/// iteration count for each cycle head. This struct tracks these cycle heads.
133#[derive(Clone, Debug, Default)]
134pub struct CycleHeads(ThinVec<CycleHead>);
135
136impl CycleHeads {
137    pub(crate) fn is_empty(&self) -> bool {
138        self.0.is_empty()
139    }
140
141    pub(crate) fn initial(database_key_index: DatabaseKeyIndex) -> Self {
142        Self(thin_vec![CycleHead {
143            database_key_index,
144            iteration_count: IterationCount::initial(),
145        }])
146    }
147
148    pub(crate) fn iter(&self) -> std::slice::Iter<'_, CycleHead> {
149        self.0.iter()
150    }
151
152    pub(crate) fn contains(&self, value: &DatabaseKeyIndex) -> bool {
153        self.into_iter()
154            .any(|head| head.database_key_index == *value)
155    }
156
157    pub(crate) fn remove(&mut self, value: &DatabaseKeyIndex) -> bool {
158        let found = self
159            .0
160            .iter()
161            .position(|&head| head.database_key_index == *value);
162        let Some(found) = found else { return false };
163        self.0.swap_remove(found);
164        true
165    }
166
167    pub(crate) fn update_iteration_count(
168        &mut self,
169        cycle_head_index: DatabaseKeyIndex,
170        new_iteration_count: IterationCount,
171    ) {
172        if let Some(cycle_head) = self
173            .0
174            .iter_mut()
175            .find(|cycle_head| cycle_head.database_key_index == cycle_head_index)
176        {
177            cycle_head.iteration_count = new_iteration_count;
178        }
179    }
180
181    #[inline]
182    pub(crate) fn push_initial(&mut self, database_key_index: DatabaseKeyIndex) {
183        if let Some(existing) = self
184            .0
185            .iter()
186            .find(|candidate| candidate.database_key_index == database_key_index)
187        {
188            assert_eq!(existing.iteration_count, IterationCount::initial());
189        } else {
190            self.0.push(CycleHead {
191                database_key_index,
192                iteration_count: IterationCount::initial(),
193            });
194        }
195    }
196
197    #[inline]
198    pub(crate) fn extend(&mut self, other: &Self) {
199        self.0.reserve(other.0.len());
200
201        for head in other {
202            if let Some(existing) = self
203                .0
204                .iter()
205                .find(|candidate| candidate.database_key_index == head.database_key_index)
206            {
207                assert_eq!(existing.iteration_count, head.iteration_count);
208            } else {
209                self.0.push(*head);
210            }
211        }
212    }
213
214    #[cfg(feature = "salsa_unstable")]
215    pub(crate) fn allocation_size(&self) -> usize {
216        std::mem::size_of_val(self.0.as_slice())
217    }
218}

250#[derive(Debug, PartialEq, Eq)]
251pub enum ProvisionalStatus {
252    Provisional { iteration: IterationCount },
253    Final { iteration: IterationCount },
254    FallbackImmediate,
255}
256
257impl ProvisionalStatus {
258    pub(crate) const fn iteration(&self) -> Option<IterationCount> {
259        match self {
260            ProvisionalStatus::Provisional { iteration } => Some(*iteration),
261            ProvisionalStatus::Final { iteration } => Some(*iteration),
262            ProvisionalStatus::FallbackImmediate => None,
263        }
264    }
